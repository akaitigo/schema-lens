# PRD: SchemaLens

## 概要
既存データベースのスキーマを解析し、正規化の不備・冗長なインデックス・外部キー欠落・命名規約違反を自動検出するスキーマ品質分析CLIツール。データのサンプリング解析によりカラムの実際の利用パターン（NULL率・カーディナリティ・データ型の適正性）を可視化し、マイグレーションSQLの自動生成まで行う。

## 背景と課題
- データベーススキーマは時間とともに劣化する（tech debt蓄積）
- 既存ツール（pganalyze等）は監視特化で、スキーマ品質の体系的分析が弱い
- 手動レビューは属人的で、網羅性に欠ける
- 「VARCHAR(255)だが実際は最大12文字」のような実態との乖離が見えない

## ターゲットユーザー
- DBA、バックエンドエンジニア、SRE
- レガシーDB移行を計画しているチーム

## 技術スタック
- Go（スキーマ解析エンジン / CLI）
- PostgreSQL / MySQL / SQLite（対応DB）

## MVP機能

### 1. スキーマ自動解析と品質スコアリング
- [x] DBに接続しスキーマ情報（テーブル・カラム・インデックス・外部キー・制約）を抽出
- [x] 正規化の不備検出（繰り返しカラム名パターン、JSONカラムの多用等）
- [x] 命名規約違反検出（snake_case / PascalCase混在等）
- [x] 型の適正性チェック（VARCHAR(255)のデフォルト利用等）
- [x] 各観点のスコアとサマリーレポート出力

### 2. カラム別データプロファイリング
- [x] 各カラムのNULL率計算
- [x] カーディナリティ（ユニーク値数 / 全行数）
- [x] 最大・最小値、平均長
- [x] データ型と実データの乖離検出

### 3. 改善提案とマイグレーションSQL自動生成
- [x] 検出した問題に対する改善提案リスト
- [x] ALTER TABLE文の自動生成
- [x] DRY RUNモード（SQLを表示するが実行しない）
- [x] 優先度付き（高・中・低）

## ユニークポイント
実データのサンプリングから「このVARCHAR(255)は最大12文字しか使われていない」「このINDEXの選択率は99%で効果なし」のような実態ベースの改善提案を生成。

## 非機能要件
- 10,000行以上のテーブルではサンプリング（デフォルト1,000行）
- 読み取り専用接続（書き込み権限不要）
- 接続文字列はDSN形式
- 出力形式: テーブル / JSON / Markdown

## 成功指標
- 3種類のDB（PostgreSQL, MySQL, SQLite）に対応
- 10テーブル以上のDBを30秒以内に解析
- 偽陽性率 < 20%
